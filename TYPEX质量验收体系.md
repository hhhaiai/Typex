## 3. 质量分层与责任边界

### 3.1 质量分层

- **L1 单元测试**：验证纯逻辑、数据结构、状态机、事务边界
- **L2 集成测试**：验证 core / platform / editor 间交互链路
- **L3 UI / E2E 测试**：验证真实浏览器中的用户旅程
- **L4 发布验收**：验证打包、Demo、文档、回归结果

### 3.2 责任边界

- `@typex/core`：数据模型、选区、事务、历史、默认动作
- `@typex/platform`：DOM、Selection、键鼠/IME 拦截、光标、平台适配
- `editor`：格式注册、工具栏、挂载、Demo 交互
- GitHub Pages：构建产物、基础可用性、主链路演示能力

---

## 4. 验收等级定义

### P0 阻塞级

必须全部通过，否则不得发布：

- 编辑器可正常加载和挂载
- 单光标输入、删除、换行正常
- 选区同步正确，无明显错位
- 撤销 / 重做可用且不破坏文档结构
- Demo 页面可访问，首屏可交互

### P1 高优先级

原则上必须通过，若失败需明确降级说明：

- 多光标创建、更新、去重正常
- IME 中文输入稳定，无重复插入/丢字
- 鼠标拖选、Shift 扩选、Alt 多选区行为符合预期
- 基础格式化命令可用
- 跨节点删除不会造成结构损坏

### P2 增强级

可在发布后迭代，但需如实标注状态：

- Markdown 双向转换
- 更复杂块级结构编辑
- 跨浏览器一致性优化
- 长文档性能基线

## 5. 测试策略

## 5.1 单元测试策略

目标：验证最小逻辑单元的正确性与边界行为。

### 优先覆盖模块

1. `History`
   - `push`
   - `undo`
   - `redo`
   - 撤销后新操作覆盖未来队列
   - 队列大小上限

2. `Selection`
   - `collapsed`
   - `rangeCount`
   - `updatePoints`
   - `_distinct` 多光标去重
   - `recoverRangesFromSnapshot`

3. 输入状态机（`initCore.js` 内输入分支）
   - 普通 `input`
   - `compositionstart`
   - `compositionend`
   - 取消输入法合成后的回滚/覆盖行为

4. 默认动作 / 事务边界
   - 删除动作触发组件删除回调
   - 非折叠选区删除
   - 事务提交后历史入栈

### 单元测试通过标准

- 核心纯逻辑模块覆盖率达到 80%+
- 高风险模块必须有边界测试，不允许只有 happy path
- 测试可重复执行，不依赖人工时序点击

---

## 5.2 集成测试策略

目标：验证系统内关键模块之间的事件链路与状态联动。

### 核心集成链路

1. **键盘输入链路**
   - `keyboardEvents` -> `input()` -> `Transaction` -> `commit()` -> `History`

2. **鼠标选区链路**
   - `mousedown` / `mouseup` -> 原生 selection -> `updateRangesFromNative()` -> caret 更新

3. **多选区链路**
   - Alt 多选区 -> `_extendRangesFromNative()` -> `_distinct()` 去重 -> UI 展示

4. **撤销重做链路**
   - `keydownz` / `keydownZ` -> `history.undo()` / `history.redo()`

5. **失焦与输入法链路**
   - `blurEvent` -> `compositionCanceled` -> compositionend 后内容状态修正

### 集成测试通过标准

- 关键事件链路必须具备断言，不只断言“无报错”
- 至少验证状态变化、选区变化、历史变化三类输出
- 对输入法和多光标场景至少覆盖 1 组异常分支

---

## 5.3 UI / E2E 测试策略

目标：在真实浏览器中验证关键用户旅程。

建议分层：

- **Smoke**：每次改动必跑，5 分钟内完成
- **Regression**：合并前/发布前跑完整回归
- **Manual Exploratory**：复杂交互做人工补充验证

### 优先自动化旅程

1. 页面打开，编辑器正常挂载
2. 点击文档，光标出现并可输入文本
3. 删除文本并验证内容变化
4. 触发撤销 / 重做并验证恢复
5. 使用工具栏进行基础格式化
6. 鼠标拖选文本并检查选区显示
7. Alt 多光标操作
8. 中文输入法输入短语并确认无重复提交
9. 刷新页面后 Demo 仍能初始化

### UI / E2E 通过标准

- P0 用户旅程 100% 通过
- P1 用户旅程通过率 >= 95%
- 不允许存在“经常复现但未定位”的 flaky 用例混入发布集
- 失败必须保留截图、控制台日志、步骤记录

## 6. GitHub Pages Demo 验证要求

README 公布的演示地址为 GitHub Pages，因此发布验收必须包含远程 Demo 验证。

### 6.1 验证目标

- 页面可访问
- 关键静态资源能加载
- 编辑器可初始化
- 基础输入能力正常
- 控件和核心交互无致命异常

### 6.2 必验清单

1. 打开 Demo 首页
2. 确认页面无空白屏
3. 确认编辑器挂载容器可见
4. 点击编辑区域，光标可见
5. 输入英文字符，内容发生变化
6. 执行删除操作，内容正确更新
7. 若支持工具栏操作，验证至少一个格式按钮有效
8. 打开浏览器控制台，确认无阻塞级报错
9. 刷新页面后功能仍可用
10. 在至少 2 个浏览器内完成 smoke 验证

### 6.3 GitHub Pages 判定标准

以下任一失败则判定 Demo 验收失败：

- 页面无法访问
- JS 资源 404 / 加载失败
- 首屏报错导致编辑器无法挂载
- 无法输入或删除
- 控制台出现阻塞级异常

## 8. 多平台与风险专项

## 8.1 浏览器维度

最低建议覆盖：

- Chrome 最新稳定版
- Safari 最新稳定版（macOS 必测）
- Firefox 最新稳定版

## 8.2 风险专项

### 富文本风险
- 跨节点删除导致结构损坏
- 选区锚点与视觉光标不一致
- 撤销后 path / selection 映射失效

### Markdown 风险
- 仅存在 UI 入口但无实际转换能力
- 格式丢失
- 块级结构导出不稳定

### 多光标风险
- 重叠光标未去重
- 多选区下删除/输入行为不一致
- 原生 selection 与内部 ranges 偏移

### 输入法风险
- compositionend 与 input 顺序问题
- blur 导致合成状态残留
- 中文输入重复插入或误删

### 多平台风险
- iframe input 在 Safari 行为差异
- 原生 selection API 差异
- 键盘修饰键在不同系统下行为不一致

## 9. 质量门禁（Quality Gates）

## Gate 1 开发完成门

进入提测前必须满足：

- 代码可构建
- 无阻塞级语法或运行时错误
- 新增/修改能力具备对应测试计划
- 自测完成并记录结果

## Gate 2 合并前门

合并前必须满足：

- P0 单元/集成/UI 用例全部通过
- 涉及核心链路变更时，回归撤销重做、选区、输入法
- 无新增高优先级缺陷未登记
- 文档与真实实现一致

## Gate 3 发布前门

发布前必须满足：

- 全量回归完成
- GitHub Pages Demo smoke 通过
- 无 P0 / P1 未解决缺陷
- 已输出最终验收报告

## Gate 4 发布后观察门

发布后 24~48 小时内观察：

- Demo 可访问性
- 控制台异常反馈
- 已知兼容性问题是否出现放大

## 10. 缺陷分级标准

### Sev-1 / P0

- 页面打不开
- 编辑器无法挂载
- 无法输入/删除
- 撤销重做导致数据严重损坏
- Demo 对外不可用

### Sev-2 / P1

- 多光标异常
- IME 输入异常
- 选区明显错位
- 常用工具栏命令无效

### Sev-3 / P2

- 个别增强能力缺失
- 低频兼容性问题
- 文案或非关键视觉异常
