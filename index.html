Developer Docs

开发文档

集成说明负责接入文档，API 文档先给介绍再进入完整参考，这样阅读顺序更自然。

1. 安装与启动

pnpm install
pnpm dev

本地启动后，你看到的默认就是产品宣传页 + 在线 Demo。先看展示，再看接入文档，会更容易形成完整认知。

2. 最小集成

import createEditor from 'editor'

const editor = createEditor({
  data: 'hello world',
})

editor
  .setToolBar(['undo', 'redo', 'header', 'color', 'bold', 'underline'])
  .mount('editor-root')

3. 当前公开 API

  • mount(id)：挂载到目标容器
  • setToolBar(items)：配置工具栏
  • getValue()：读取当前文档
  • setValue(data)：替换当前文档
  • execCommand(name, value)：执行命令

4. 推荐接入流程

  • 先用字符串或 legacy 文档初始化，确认挂载和输入链路正常
  • 再接入 toolbar、change 监听和右侧预览
  • 如果需要结构化输出，可直接传 canonical document
  • 复杂扩展优先看 API 文档，再结合仓库里的 Demo wiring

API 介绍文档

这里先说明 API 文档怎么读：先理解产品能力和接入方式，再进入方法、命令与数据结构详情。这样更容易把 Typex 的边界和使用方式看清楚。

核心公开能力

  • mount(id)：挂载编辑器
  • setToolBar(items)：设置工具栏
  • getValue()：获取当前文档
  • setValue(data)：写入新内容
  • execCommand(name, value)：执行编辑命令

建议阅读顺序
